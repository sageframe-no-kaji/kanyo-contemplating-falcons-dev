Add New Stream

Creates the on-disk stream definition: directory, config.yaml, clips/ and logs/. Starting a detection container stays a manual step (shown after creation).

Stream Identity

Lowercase letters, numbers, hyphens, underscores only. Becomes the directory (kanyo-<id>) and container name (kanyo-<id>-gpu).

What "Create Stream" does:

  1. Creates /opt/services/kanyo-<id>/ with clips/ and logs/
  2. Writes config.yaml from the config template plus your values
  3. The stream appears in the overview immediately (auto-discovery)

What it deliberately does NOT do:

  • Start a detection container (manual compose paste — instructions shown after creation)
  • Create ZFS datasets (needs root on the host)
  • Create the Telegram channel or add the bot as admin (Telegram-app-only steps)
